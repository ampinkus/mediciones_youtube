
## Información obtenible de un video de YouTube mediante su API (YouTube Data API v3)

Este documento detalla la información que se puede obtener de un video o stream en YouTube mediante el uso del endpoint `videos` de la API de YouTube, junto con una explicación detallada de su posible utilidad en un sistema de medición como el Tablero de Vistas.

---

### 🔗 Endpoint principal: `videos`
Permite recuperar información detallada sobre uno o varios videos públicos.

Parámetro clave: `part`, que define los bloques de información a obtener.

---

### 1. `snippet` - Información descriptiva del video

| Campo            | Descripción                                                                 | Uso sugerido                                                                                                 |
|------------------|-----------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------|
| `title`         | Título del video                                                            | Mostrar como nombre principal en la tabla o tarjeta del tablero.                                              |
| `description`   | Descripción del video                                                       | Tooltip o vista expandida con más contexto sobre el video.                                                    |
| `publishedAt`   | Fecha y hora de publicación                                                 | Permite analizar la evolución del canal en el tiempo y filtrar por fechas.                                    |
| `channelId`     | ID del canal que subió el video                                             | Útil para relacionar múltiples videos con el mismo canal.                                                     |
| `channelTitle`  | Nombre del canal                                                            | Mostrar en el tablero como entidad principal (fuente del contenido).                                          |
| `tags[]`        | Etiquetas asignadas por el autor                                            | Permite categorizar y agrupar videos por temas de interés.                                                    |
| `categoryId`    | ID de categoría (ej: música, noticias, etc.)                               | Clasificación general útil para segmentar por tipo de contenido.                                              |
| `thumbnails`    | URLs de miniaturas (default, medium, high)                                  | Mostrar como imagen de vista previa, mejora visual del tablero.                                               |

---












### 2. `statistics` - Métricas cuantitativas del video

| Campo            | Descripción                                                                 | Uso sugerido                                                                                                 |
|------------------|-----------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------|
| `viewCount`     | Número total de vistas                                                      | Métrica principal para evaluar el alcance del video.                                                          |
| `likeCount`     | Número de "me gusta"                                                        | Permite medir la aceptación del video. Se puede comparar con las vistas para medir engagement.               |
| `favoriteCount` | No se usa, siempre es 0                                                     | No relevante actualmente.                                                                                     |
| `commentCount`  | Cantidad de comentarios                                                     | Refleja el nivel de interacción del público con el contenido.                                                 |

---

### 3. `liveStreamingDetails` - Detalles de transmisiones en vivo
Solo disponible para videos en vivo o que fueron en vivo.

| Campo                  | Descripción                                                         | Uso sugerido                                                                                             |
|------------------------|---------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------|
| `actualStartTime`     | Hora real de inicio del stream                                      | Medir duración real del evento.                                                                            |
| `actualEndTime`       | Hora real de finalización                                           | Igual al anterior. Permite calcular duración total.                                                        |
| `scheduledStartTime`  | Hora programada para iniciar                                        | Comparar programación vs ejecución real.                                                                   |
| `scheduledEndTime`    | Hora programada de fin                                              | -                                                                                                         |
| `concurrentViewers`   | Vistas simultáneas (solo disponible durante la transmisión activa) | Muy útil para analizar el pico de audiencia y nivel de interés en tiempo real.                             |

---

### 4. `contentDetails` - Características técnicas del video

| Campo             | Descripción                                                   | Uso sugerido                                                                                      |
|-------------------|---------------------------------------------------------------|----------------------------------------------------------------------------------------------------|
| `duration`        | Duración del video en formato ISO 8601 (ej: `PT3M45S`)       | Convertir a minutos o segundos para representar duración media de los contenidos.                 |
| `dimension`       | 2D o 3D                                                       | Detectar si es contenido especial (opcional).                                                     |
| `definition`      | Calidad del video (sd, hd)                                   | Verificar calidad del contenido, puede afectar experiencia del usuario.                          |
| `caption`         | Indica si tiene subtítulos                                   | Importante para accesibilidad, o filtrado de contenidos subtitulados.                             |
| `licensedContent` | Si el video tiene contenido con licencia                     | Permite detectar si el video está sujeto a restricciones de copyright.                            |
| `projection`      | Tipo de proyección (rectangular, 360°, etc.)                | Relevante para detectar experiencias inmersivas.                                                  |

---

### 5. `player` - Código para embeber el video

| Campo         | Descripción                                   | Uso sugerido                                                            |
|---------------|-----------------------------------------------|-------------------------------------------------------------------------|
| `embedHtml`   | Código HTML para embeber el reproductor       | Incrustar el video directamente en el tablero para vista rápida.       |

---

### 6. `status` - Estado de publicación del video

| Campo                 | Descripción                                             | Uso sugerido                                                                                   |
|-----------------------|---------------------------------------------------------|-----------------------------------------------------------------------------------------------|
| `uploadStatus`        | Estado del procesamiento (ej: processed, rejected)     | Filtrar videos correctamente subidos.                                                         |
| `privacyStatus`       | Público, oculto, privado                               | Filtrar solo videos públicos visibles al usuario.                                             |
| `license`             | Tipo de licencia (standard o Creative Commons)         | Relevante para posibles reutilizaciones.                                                       |
| `embeddable`          | Si se puede embeber                                    | Evitar errores al intentar incrustar videos no embebibles.                                    |
| `publicStatsViewable` | Si las estadísticas son visibles al público           | Asegura que `viewCount` y similares sean confiables.                                          |

---

### 7. `recordingDetails` - Información de grabación (si está disponible)

| Campo                | Descripción                               | Uso sugerido                                                           |
|----------------------|-------------------------------------------|------------------------------------------------------------------------|
| `recordingDate`      | Fecha en que fue grabado el video        | Distinguir entre fecha de grabación y publicación.                     |
| `location`           | Latitud y longitud de grabación          | Crear visualizaciones geográficas o mapas.                             |
| `locationDescription`| Descripción textual del lugar            | Mostrar como información contextual adicional del video.               |

---

### 📌 Recomendación para tu sistema de medición:
Usar al menos esta combinación:
```
part=snippet,statistics,liveStreamingDetails,contentDetails
```
Y si querés incrustar el video:
```
+player
```

Esto te garantiza acceso a información útil tanto para métricas como para visualización en tu tablero.
